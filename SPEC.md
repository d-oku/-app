# 音声文字起こし・要約アプリ 仕様書 (SPEC.md)

## 1. アプリケーション概要
本アプリケーションは、ブラウザ上（HTML/CSS/JSのみ）で動作し、アップロードされた音声ファイルから「文字起こし」「要約」「要点抽出」を自動で行うシングルページアプリケーション（SPA）です。バックエンドサーバーを持たず、すべての通信は直接Gemini APIに対して行われます。スマートフォンでの利用を想定したレスポンシブデザイン（モバイルファースト）を採用します。

## 2. 技術スタック
- **構造・ロジック**: HTML5, Vanilla JavaScript (ES6+)
- **スタイリング**: Vanilla CSS (CSS Variables, Flexbox/Gridを活用し、モダンでリッチなUIを構築)
- **外部API**: Google Gemini API (モデル: `gemini-1.5-pro` または `gemini-1.5-flash`)
- **データ保存**: `localStorage` (APIキーの保存に使用)

## 3. Gemini API 仕様と制約事項
最新の公式ドキュメントに基づく音声処理の仕様は以下の通りです。

- **対応フォーマット**: WAV, MP3, AIFF, AAC, OGG, FLAC
- **サイズ制限**:
  - **Inline Data (Base64エンコード)**: APIリクエストに直接含める場合、リクエスト全体で最大 **100MB**。ただし、ブラウザ上でのBase64変換の負荷を考慮し、数MB〜十数MB程度の短い音声に適しています。
  - **File API**: より大きなファイルの場合、File APIエンドポイント (`/upload/v1beta/files`) にアップロードすることで最大 **2GB** まで対応可能です（ファイルは48時間保持されます）。本アプリでは、汎用性と安定性を考慮し、ファイルサイズに応じてInline DataまたはFile APIを使い分けるか、長時間の音声向けにFile APIへのアップロードを基本実装とします。
- **音声処理の仕様**:
  - 音声は 16 Kbps のデータ解像度にダウンサンプリングされます。
  - マルチチャンネル（ステレオなど）は単一チャンネル（モノラル）に結合されて処理されます。
  - 1回のプロンプトに含まれる音声の合計長は最大 9.5 時間までです。

## 4. 画面構成とUIフロー
モバイルでの使いやすさを重視した1画面（SPA）構成とし、セクションを切り替えて表示します。

### 4.1. 初期設定 / 設定モーダル
- **要素**: APIキー入力用テキストフィールド、保存ボタン、APIキー取得方法のリンク。
- **挙動**: 
  - 初回アクセス時、または `localStorage` にAPIキーが存在しない場合は設定モーダルを強制表示。
  - 入力されたキーは `localStorage` に保存され、次回以降は自動で読み込まれます。
  - 右上の「設定⚙️」アイコンからいつでも変更可能。

### 4.2. メイン画面（アップロード＆アクション）
- **要素**: 
  - ファイルドロップ＆選択エリア（タップでファイルエクスプローラー起動）
  - 選択されたファイル名・サイズ・形式の表示
  - 「文字起こし・要約を開始する」ボタン（ローディングアニメーション付き）
- **挙動**:
  - 対応していない形式や上限サイズを超えるファイルが選択された場合は即座に警告を出します。

### 4.3. 結果表示画面
- **要素**: 
  - タブUI（「文字起こし」「要約」「要点抽出」）
  - 結果を表示するテキストエリア
  - 「クリップボードにコピー」ボタン、「最初からやり直す」ボタン
- **挙動**:
  - 処理完了後、滑らかなトランジションで結果画面を表示。
  - タブを切り替えることで、用途に応じたテキストを閲覧・コピー可能。

## 5. エラー処理 (Error Handling)
ユーザーのフラストレーションを軽減するため、適切なエラーメッセージとリカバリ手順を提示します。

1. **APIキーエラー (400, 401, 403)**:
   - 「APIキーが無効、または設定されていません。設定画面から正しいキーを入力してください。」
   - 自動的に設定モーダルを展開。
2. **ファイル形式・サイズエラー**:
   - 「サポートされていないファイル形式です（対応: WAV, MP3, AAC, OGG, FLAC）。」
   - 「ファイルサイズが大きすぎます（上限: 〇〇MB）。」※実装するAPI方式に応じた上限値を提示。
3. **通信エラー・タイムアウト**:
   - 「ネットワークエラーが発生しました。インターネット接続を確認して再度お試しください。」
4. **APIレート制限 (429)**:
   - 「APIのリクエスト上限に達しました。しばらく待ってから再度お試しください。」
5. **汎用処理エラー**:
   - 「音声の解析に失敗しました。ノイズが多いか、無音の可能性があります。」

## 6. モダンUI/UXデザインの要件
- **カラーパレット**: ダークモードを基調とした洗練されたデザイン（例: 深いネイビーグレーの背景に、アクセントカラーとして鮮やかなパープルやブルーのグラデーション）。
- **タイポグラフィ**: `Inter` や `Roboto` などの視認性の高いサンセリフ体を使用。
- **マイクロインタラクション**: 
  - ボタンのホバー時やタップ時にスケールやシャドウが変化する。
  - ファイル処理中（解析中）は、ただのスピナーではなく、音声波形を模した動的なローディングアニメーションを表示し、ユーザーを飽きさせない工夫をする。
- **グラスモーフィズム**: モーダルやカード要素に半透明の背景と背景ぼかし（`backdrop-filter`）を適用し、プレミアム感を演出。

## 7. 今後の拡張性（オプション）
- 処理結果をMarkdownやTXTファイルとしてダウンロードする機能。
- 過去の履歴をIndexedDBに保存し、後から見返せる機能。
