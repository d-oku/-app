# 開発プロンプト集 (PROMPTS.md)

本プロジェクトの開発において、AIエージェントの精度を引き出し、課題を解決するために投げた主要なプロンプトの記録です。

### 1. 最小構成（MVP）の要件定義と仕様策定
> 「ブラウザのHTML/CSS/JSのみを使って、音声ファイルから文字起こし・要約・要点抽出を行うスマホ対応のWebアプリを作ります。APIはGemini APIを使い、キーはlocalStorageに保存します。この要件をもとに、最新のGemini APIの公式ドキュメントを検索し、画面の流れやエラー処理を含めた `SPEC.md` を作成してください。」

### 2. セキュリティを担保した認証ロジックの実装
> 「APIキーを安全に管理するための認証ロジックとUIを実装してください。アプリ起動時に `localStorage.getItem('gemini_api_key')` を確認し、ない場合は入力モーダルを表示、ある場合はメイン画面を表示するようにしてください。GitHubの公開リポジトリにキーが残らないようにすることが必須です。」

### 3. APIリクエストの最適化（JSON出力モード）
> 「提供された音声ファイルから『transcript（文字起こし）』『summary（3行要約）』『keypoints（3〜5個の要点）』の3つを抽出し、必ず指定されたJSON形式で出力するようにプロンプトと設定（`responseMimeType: "application/json"`）を更新してください。」

### 4. トラブルシューティング①：APIモデルのエラー対応
> 「現状、ファイルをアップロードすると『models/gemini-1.5-flash is not found for API version v1beta...』というエラーが起きます。使われているgemini-1.5-flashではなく、最新のgemini-3.5-flashに変更してください。」
> **→ 結果:** AIがエンドポイントURLを最新モデルに書き換え、正常な通信を確立。

### 5. トラブルシューティング②：UI崩壊とファイルアップロードバグの修正
> 「Uncaught ReferenceError: ui is not defined at app.js:2:1 というエラーが起きていました。また、スーパーリロードを行うとファイルをアップロード、ドラッグアンドドロップをする画面すら無くなってしまいました。PCでもスマホ（パターンA）でもファイル選択画面が開きません。改善をお願いします。」
> **→ 結果:** AIが部分的な手作業の修正による「HTML/JSの構造崩壊」と「hidden属性によるブラウザのセキュリティブロック」を特定。`display: none;` や `.m4a` 対応を含めた**完全版のコードとして丸ごと再生成**させることで、すべてのバグを一掃した。
