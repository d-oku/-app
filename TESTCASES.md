# テスト仕様書 (TESTCASES.md)

本ドキュメントは、音声解析アシスタント（Audio Analyst）の動作確認・受け入れテスト（UAT）の観点と結果です。

## 1. 正常系テスト（Happy Path）
| テスト項目 | 操作手順 | 期待される結果 |
| :--- | :--- | :--- |
| **スマホからのファイル投入** | iPhone等のブラウザでアクセスし、画面中央をタップしてボイスメモ（.m4a等）を選択。 | スマホ標準のファイル選択が起動し、選択後にファイル名とサイズが表示され、解析ボタンがアクティブになること。 |
| **解析の正常完了とUI遷移** | 15MB以内の音声を選択し「解析を開始する」をタップ。 | ローディングUIが表示された後、結果画面へ自動でスクロールし、「文字起こし」タブに全文が表示されること。 |
| **タブ切り替えと要約・要点** | 解析完了後、「要約」および「要点抽出」タブをタップする。 | 選択したタブがハイライトされ、3行要約と箇条書きの要点がそれぞれ正しく表示されること。 |
| **クリップボードへのコピー** | 任意のタブを開いた状態で「コピー」ボタンをタップ。 | ボタンのテキストが「✔ コピー完了」に2秒間変化し、端末のクリップボードにテキストが保存されること。 |
| **状態の初期化（リセット）** | 「やり直す」ボタンをタップ。 | 選択ファイルとテキストエリアがクリアされ、初期のアップロード画面に戻ること。 |

## 2. 異常系テスト（Error Handling）
| テスト項目 | 操作手順 | 期待される結果 |
| :--- | :--- | :--- |
| **APIキー未設定での実行** | 初回起動時、APIキーを空のままモーダルを閉じ、ファイルを選んで解析を実行。 | 「APIキーが設定されていません」とエラーが表示され、再度設定モーダルが開くこと。 |
| **無効なAPIキーでの通信** | 設定から無効な文字列を保存し、解析を実行。 | 通信後に「APIエラー: APIキーが無効です。設定を確認してください。」と表示されること。 |
| **サイズ超過ファイルのブロック** | 15MBを超える音声ファイルを選択。 | 「ファイルサイズが大きすぎます（上限: 15MB）」と即座にエラーが表示され、解析ボタンが無効化されること。 |
| **セキュリティブロックの回避** | PCおよびスマホブラウザで、アップロードエリアをタップ。 | `hidden` 属性によるブラウザのクリック無効化が起きず、確実にファイルピッカーが開くこと（バグ修正確認）。 |
